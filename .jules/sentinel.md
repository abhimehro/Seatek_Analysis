## 2025-07-21 - Subprocess Environment Exfiltration via Dictionary Merging
**Vulnerability:** The script manually merged external user-provided overrides (`custom_env`) into a filtered environment dictionary `filtered_env`, and *then* applied an explicit denylist (`GH_TOKEN`, `GITHUB_TOKEN`). If an explicit credential was provided in `custom_env`, it was correctly allowed *unless* it matched the denylist. However, if a heuristic (like filtering out keys containing `TOKEN` or `SECRET`) is added *after* the `custom_env` is applied, it will mistakenly strip explicitly provided credentials required by the automation scripts. More dangerously, if the heuristic runs *before* the overrides but is poorly implemented, implicit secrets might bleed through.
**Learning:** Defense-in-depth environment variable filtering requires order-of-operations awareness. Heuristics meant to prevent implicit "environment bleed" from the runner must run on the base environment *before* user-specified overrides are applied, so that the overrides are trusted and not accidentally stripped.
**Prevention:** When applying heuristic filters to prevent credential exfiltration in `subprocess` environments, apply the heuristic to the base environment, *then* merge explicitly provided custom overrides, and finally apply any hardcoded strict denylists as a last line of defense.
